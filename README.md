# 🐶 양구 애견동반 판별기 MVP

> **양구군 반려견 동반 여행 정보를 30초 안에 확인할 수 있는 No-AD, Pure Data 기반 정적 웹 유틸리티**

본 프로젝트는 양구군을 방문하려는 반려견 보호자가 반려견 크기, 여행 목적, 우선순위를 선택하면 양구군 내 반려동물 동반 가능 후보 장소와 방문 전 확인사항을 30초 안에 확인할 수 있도록 돕는 공익형 웹 유틸리티입니다. 광고, 회원가입, 예약, 결제, 커뮤니티 기능을 완전히 배제한 정적 웹 서비스입니다.

---

## 1. 프로젝트 개요
- **서비스명**: 양구 애견동반 판별기 MVP
- **서비스 목적**: 수도권 자차 이동객 및 당일치기/1박2일 양구 방문 보호자들의 정보 접근성 개선
- **개발 원칙**: 
  - **No Framework**: React, Vue, Tailwind CSS 등 외부 빌드 도구 및 라이브러리 일절 배제
  - **No Backend**: 서버 및 DB 인프라 없는 완전 정적 서버리스(Static First) 구조
  - **No-AD, Pure Data**: 유료 배너, 제휴 마케팅, 애드센스 전면 미지원
  - **개인정보 최소화**: 로그인, 위치수집, 반려견 정보 저장이 없는 순수 데이터 조회용 유틸리티

## 2. 실행 방법

본 프로젝트는 빌드 도구나 웹팩 등의 설정이 전혀 없는 정적 웹 앱이므로 매우 간편하게 구동할 수 있습니다.

### 방법 A. 로컬 정적 서버 구동 (권장)
비동기 `fetch()` API의 로컬 보안 보안 제한(CORS)을 준수하기 위해 로컬 웹 서버 구동을 권장합니다.

1. VS Code 등의 에디터를 실행한 후 `Live Server` 확장을 설치하여 구동합니다.
2. 또는 터미널(VS Code / PowerShell 등)을 열어 아래와 같이 Python 혹은 Node.js 간이 서버를 실행합니다.

```bash
# Python이 설치된 경우
python -m http.server 8000

# Node.js (npx)가 설치된 경우
npx serve .
```

실행 후 브라우저에서 `http://localhost:8000` 혹은 제공된 포트로 접속합니다.

### 방법 B. 로컬 파일 직접 열기
인터넷 브라우저의 종류(Safari, Firefox 일부 구버전 및 브라우저 보안 설정을 변경한 경우)에 따라 `index.html` 파일을 더블클릭하여 바로 오픈하여 테스트해 볼 수도 있습니다. (단, Chrome 및 최신 Edge 등은 비동기 fetch 보안 정책으로 인해 정적 로컬 서버 사용을 강력하게 권장합니다.)

## 4. 데이터 수정 방법

양구군 내의 반려동물 동반 가능 신규 후보 장소를 추가하거나, 기존 장소의 번호 또는 주소가 변경되었을 경우 별도의 코드를 만질 필요 없이 `/data/places.json` 파일의 배열 구조에 맞춰 객체를 수정하시면 화면에 실시간 자동 반영됩니다.

### 데이터 수정 및 추가 예시
`/data/places.json`을 텍스트 에디터로 열어 배열 내부에 규격화된 포맷으로 장소 정보를 수정하거나 추가합니다.
```json
  {
    "id": "yanggu-011",
    "name": "새로운 양구 애견동반 카페",
    "category": "카페·식당",
    "address": "강원특별자치도 양구군 양구읍 중앙로 999",
    "phone": "033-481-9999",
    "petSize": ["소형견", "중형견"],
    "purposeTags": ["카페·식당"],
    "trustGrade": "B",
    "petNote": "야외 정원에서만 소중형견 동반이 가능합니다. 주말 방문 시 대기가 발생할 수 있으니 사전 예약을 고려하세요.",
    "sourceName": "사용자 커뮤니티 제보",
    "sourceUrl": "",
    "officialUrl": "https://example.com/cafe",
    "lastUpdated": "2026-06-02",
    "lat": 38.1052,
    "lng": 127.9822
  }
```

## 5. places.json 필드 설명

| 필드명 | 타입 | 필수 여부 | 설명 |
| :--- | :--- | :---: | :--- |
| `id` | `string` | 필수 | 장소를 구분하는 고유 식별 코드 (예: `yanggu-001`) |
| `name` | `string` | 필수 | 장소의 한글 정식 명칭 |
| `category` | `string` | 필수 | 화면 좌측 상단 표시 카테고리 (`관광지`, `산책`, `카페·식당`, `숙박`) |
| `address` | `string` | 필수 | 도로명 혹은 지번 주소 |
| `phone` | `string` | 선택 | 문의 가능한 전화번호 (생략 시 전화하기 버튼 비활성화) |
| `petSize` | `array` | 필수 | 허용되는 반려견 크기 조합 (`"소형견"`, `"중형견"`, `"대형견"`) |
| `purposeTags` | `array` | 필수 | 탐색 목적 필터링 매칭 태그 (`"산책"`, `"관광지"`, `"카페·식당"`, `"숙박"`) |
| `trustGrade` | `string` | 필수 | 데이터 정확성 기준 신뢰 등급 (`"A"`: 공식 정보, `"B"`: 방문 전 확인 권장, `"C"`: 유저 참고용) |
| `petNote` | `string` | 선택 | 반려동물 동반 안내 부가 설명 문구 |
| `sourceName` | `string` | 필수 | 수집 데이터의 출처 기관 또는 경로 명칭 |
| `sourceUrl` | `string` | 선택 | 출처 원본 정보 사이트 주소 |
| `officialUrl` | `string` | 선택 | 대상 영업소 또는 관광지 공식 사이트 주소 (생략 시 홈페이지 버튼 비활성화) |
| `lastUpdated` | `string` | 필수 | 데이터의 최종 조사/갱신 날짜 (`YYYY-MM-DD`) |
| `lat` | `number` | 선택 | 지도 포털 좌표 매칭을 위한 위도 (Latitude) |
| `lng` | `number` | 선택 | 지도 포털 좌표 매칭을 위한 경도 (Longitude) |

## 6. 배포 방법

정적 호스팅 서비스를 통해 1분 만에 전 세계에 무료 배포할 수 있습니다.

### GitHub Pages 배포 (우선 권장)
1. GitHub 리포지토리를 새로 생성합니다.
2. 로컬 작업 폴더 내 모든 파일을 업로드합니다.
3. 리포지토리의 `Settings` -> `Pages` 탭으로 이동합니다.
4. `Build and deployment` 섹션에서 `Branch`를 `main` 또는 `master` 브라우저의 `/root` 경로로 지정하고 `Save`를 누릅니다.
5. 잠시 후 상단에 표시되는 `https://<유저네임>.github.io/<리포지토리명>/` 링크로 배포가 자동 발행됩니다.

### Vercel / Netlify / Cloudflare Pages 배포
각 플랫폼 웹사이트에 접속하여 GitHub 리포지토리를 연결해 주시면 자동으로 어떠한 빌드 설정 없이 정적 빌드로 퍼블리싱을 안전하게 진행해 줍니다.

## 8. 향후 개선 가능 항목

본 버전은 MVP 제품으로 향후 아래와 같은 방향으로 자유롭게 기능 고도화가 가능합니다.
1. **PWA(Progressive Web App) 도입**: 오프라인 상태에서도 양구군 산 속에 전파가 통하지 않을 때 저장된 `places.json`을 캐싱하여 조회가 가능한 네이티브 앱 형태로 진화 가능.
2. **지도 SDK 연동**: 추후 양구군의 관광 수요가 급증하는 경우 네이버 지도 API 혹은 카카오 맵 SDK를 발급받아 화면 내부에 개별 핀으로 여러 군데를 한눈에 지도 뷰로 띄워주는 지도 기반 탐색 탭 연동 가능.
3. **사용자 동반 장소 제보 서식 추가**: 이메일 API나 간단한 Google Form 링크 처리를 통하여 신규 장소를 접수하는 채널 마련.
